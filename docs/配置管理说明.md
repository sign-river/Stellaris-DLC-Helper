**配置管理说明**
- 位置：设置 → 配置管理 选项卡。
- 作用：显示当前程序生效的 `config.json` 路径，便于诊断、复制或在资源管理器中打开该目录进行替换。
- 使用说明：
  - `config.json` 已包含在程序发布包中，可直接使用。
  - 打包模式：程序优先读取 EXE 同目录的 `config.json`。
  - 开发模式：程序优先读取项目根目录（模块目录）或当前工作目录的 `config.json`。
  - 如需修改配置（如更换服务器地址），请直接编辑对应目录下的 `config.json` 并重启程序。
  - 如出现"仍然使用旧配置"的问题，请打开"设置 → 配置管理"，复制并发送 `config.json` 路径或将该目录下的 `runtime_info.txt` 与日志一起提交给开发者进行诊断。
- 备注：程序启动时也会在相应目录写入 `runtime_info.txt`，包含 `sys.executable`、`cwd`、`config_path`、`DLC_SERVER_URL` 等信息，便于快速定位问题。
